## 🧩 我的工程观

我很重视工程实践中的“可复现、可解释、可交接”。

相比只把功能做出来，我更在意：

- 部署过程是否稳定
- 故障是否能快速定位
- 文档是否能让团队成员直接接手
- 自动化是否真的减轻重复劳动

在日常工作里，我通常会优先考虑：

- Docker / Docker Compose 标准化部署
- 通过脚本减少人工操作
- 用日志、文档和排障记录沉淀经验
- 把临时解决方案逐步升级成长期可维护方案

## 🌊 Aquaflow（水流计划）

这是我非常重视、也会长期投入的一项公益行动。

Aquaflow 的核心方式，是通过 City Walk 的形式，在北京不同公园、街区和工作地带徒步，为顶着烈日工作的环卫工人、建筑工人和安保人员送去饮用水。

我不只是临时参与，而是把这件事当作一个长期行动来执行：

- 设计路线
- 记录行动过程
- 沉淀执行日志
- 复盘怎样触达更精准、效率更高

我希望这件事既保留温度，也能形成真正可持续的行动方法。
