第一门：线性代数（Week 1–4）

Week 1 — 向量基础

向量 = 一列有序数字。AI 里每个词、每张图都被表示成向量（Embedding）。

a = [1, 2, 3]    b = [4, 5, 6]

点积（Dot Product）:
a · b = 1×4 + 2×5 + 3×6 = 32
→ Attention 机制本质就是做点积，判断两个词的"相关度"

余弦相似度（向量数据库 RAG 用这个找最相似文档）:
cos(θ) = (a · b) / (|a| × |b|)
|a| = √(1²+2²+3²) = √14 ≈ 3.74

练习：给定 q=[1,0,1]，k=[1,1,0]，计算点积和余弦相似度。

Week 2 — 矩阵乘法（最重要！）

矩阵 = 向量的集合，二维数据表。把矩阵想象成"数据变换器"——输入向量，输出新向量。

A (2×3) × B (3×2) = C (2×2)
C[0][0] = 行0·列0 = 1×7 + 2×9 + 3×11 = 58

理解 Self-Attention：
Attention(Q,K,V) = softmax(QKᵀ / √d) × V
  1. QKᵀ  → 矩阵乘法，算"每对词的相关度分数"
  2. / √d  → 防止数值过大（d 是向量维度）
  3. softmax → 分数变概率（概率之和=1）
  4. × V  → 用概率加权汇聚 V，得到最终表示

Week 3 — 矩阵分解（理解 LoRA）

大模型权重矩阵 4096×4096 ≈ 1600万参数。LoRA 用两个小矩阵代替，大幅压缩训练量：

原始更新：ΔW (4096×4096) → 16,777,216 参数
LoRA：   A (4096×8) × B (8×4096) → 只需 65,536 参数！节省 256 倍

原理：大多数权重更新是"低秩"的——用小矩阵近似大矩阵，
      损失极少精度，但训练速度和显存消耗大幅下降。

Week 4 — Python NumPy 实战

import numpy as np

# 向量点积
q = np.array([1, 0, 1])
k = np.array([1, 1, 0])
print(np.dot(q, k))          # → 1

# 矩阵乘法 (Self-Attention 简化版)
Q = np.random.randn(4, 3)    # 4个词，每词3维
K = np.random.randn(4, 3)
scores = Q @ K.T              # (4×4) attention分数矩阵
print(scores.shape)           # (4, 4)

第二门：微积分 · 梯度下降（Week 5–7）

Week 5 — 导数与梯度下降

AI 训练的核心目标：找让"损失函数"最小的参数。

梯度下降（像闭眼下山）：
  1. 感受脚下坡度 → 计算梯度（偏导数）
  2. 朝最陡下坡方向迈一步 → 更新参数
  3. 重复，直到谷底 → 损失最小

公式：θ_new = θ_old - lr × ∂L/∂θ
                          ↑       ↑      ↑
                        新参数  学习率  梯度

偏导数示例：
L(w₁, w₂) = w₁² + 2w₁w₂ + w₂²
∂L/∂w₁ = 2w₁ + 2w₂   (把 w₂ 当常数，对 w₁ 求导)
∂L/∂w₂ = 2w₁ + 2w₂   (把 w₁ 当常数，对 w₂ 求导)

Week 6 — 链式法则（反向传播的核心）

链式法则：y = f(g(x))  →  dy/dx = f'(g(x)) × g'(x)

例：y = (x²+1)³
令 u = x²+1，则 y = u³
dy/dx = 3u² × 2x = 3(x²+1)² × 2x

神经网络反向传播 = 链式法则的连续应用：
  损失 L → 最后一层 → 倒数第二层 → ... → 第一层
  每层都用链式法则把梯度"传回去"

实际使用 PyTorch，它自动计算（autograd），不需要手算。

Week 7 — AI 常用函数速查

函数导数AI 用途
eˣeˣSoftmax 分母
ln(x)1/x交叉熵损失
max(0, x)0 或 1ReLU 激活函数
1/(1+e⁻ˣ)σ(x)(1-σ(x))Sigmoid（二分类）

第三门：概率论与统计（Week 8–9）

Week 8 — Softmax 与交叉熵损失

Softmax：把模型输出的原始分数变成概率分布（所有概率之和 = 1）。

模型输出（logits）：["你好"→3.2, "再见"→1.1, "谢谢"→2.8]

P("你好") = e³·² / (e³·² + e¹·¹ + e²·⁸)
           = 24.5 / (24.5 + 3.0 + 16.4) = 55.7%

交叉熵损失（衡量预测准不准）：
  预测概率 P = [0.7, 0.2, 0.1]，正确答案是第一个
  Loss = -log(0.7) = 0.357   ← 预测越准，Loss 越小
  若 P[0]=0.01，Loss = -log(0.01) = 4.6  ← 预测很差

Week 9 — 期望与 KL 散度

期望（加权平均）：
E[X] = Σ x × P(x)
→ RLHF 的 Reward Model 估计"这个回答的期望质量"

KL 散度（衡量两个概率分布的距离）：
KL(P||Q) = Σ P(x) × log(P(x)/Q(x))
KL = 0 → 两分布完全相同
KL 越大 → 差异越大

→ DPO/RLHF 训练时用来防止模型偏离原始分布太远
   （防止模型"学歪"，忘掉原来的语言能力）
